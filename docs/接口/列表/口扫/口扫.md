# 接口名称
口扫
# 接口描述
该接口用于口扫上下颌，将输入的口扫上下颌模型进行AI分析计算输出关键点位置和KPI结果。
# 请求URL
`POST /engine/v2/mesh/oral-scan`
# 请求头
需要在请求头中加入鉴权信息
# 回调地址
# 输入定义


| 参数               | 类型     | 说明                                |
| ---------------- | ------ | --------------------------------- |
| sid              | String | 唯一ID可以传入任意String，用于回调查询等操作，请确保唯一性 |
| rawData.lowerUrl | String | 口扫下颌模型URL，请保证URL为STL模型的URL且可用     |
| rawData.upperUrl | String | 口扫上颌模型URL，请保证URL为STL模型的URL且可用     |
| callback         | Object | [回调](/入门/回调)                      |
# 输入示例
```json
{  
  "sid": "{{$guid}}",  
  "rawData": {  
    "lowerUrl": "https:XXXXXXXXX.stl",  
    "upperUrl": "https:XXXXXXXX.stl"  
  },  
  "callback": [  
    {  
      "type": "rabbitmq",  
      "out": "aaaaa"  
    }  
  ]  
}
````


# 输出定义
| 参数                      | 类型     | 说明                                |
| ----------------------- | ------ | --------------------------------- |
| sid                     | String | 唯一ID可以传入任意String，用于回调查询等操作，请确保唯一性 |
| code                    | Int    | 接口返回code码，HTTP状态码                 |
| errMsg                  | String | 接口错误信息                            |
| result                  | Object | 返回的详细数据对象                         |
| result.rawData          | Object | 口扫上 下颌文件URL                       |
| callback                | Object | [回调](/入门/回调)                      |
| metadata.rectify_stl    | Object | 分割出来 的文件(单颗牙和整颌)的存储地址(URL)        |
| metadata.matrix4x4      | Arry   | 口扫文件的转正矩阵                         |
| metadata.screenshotList | Object | 口扫模型不同视角截图                        |
| metadata.landmarksFrom  | Arry   | 用于计算旋转矩阵的关键点位坐标(from和to共同计算)      |
| metadata.landmarksTo    | Arry   | 用于计算旋转矩阵的关键点位坐标(from和to共同计算)      |
| result.landmark         | Object | 计算出来的点 坐标                         |
| result.kpi              | Object | 根据点结合医学计算出来的指标                    |
